# Prisma Rules - VCANews

Guidelines for safe and efficient database interactions.

## Access Patterns
* **Repository Exclusive**: All database access must be encapsulated in the `/repositories` directory.
* **Forbidden**: No direct `prisma` calls in components, API routes, or services.

## Query Best Practices
* **Selective Fetching**: Use `select` to retrieve only necessary fields. Avoid `include` unless relations are explicitly required.
* **Pagination**: Mandatory for all list endpoints. Use `skip` and `take`.
* **Large Relations**: Avoid fetching large nested arrays in a single query; prefer specialized repository methods.
* **Raw SQL**: Avoid raw SQL unless performance requires it; if used, perform strict validation on inputs.

## Type Safety
* **Generated Types**: Always use types generated by Prisma (`Prisma.XCreateInput`, `Prisma.XUpdateInput`).
* **Prisma Client**: Use the centralized client exported from `@/lib/db`.
